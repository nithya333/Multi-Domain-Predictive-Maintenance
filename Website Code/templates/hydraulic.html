Details about the Model

Here's how your data is converted into useful insights!

Model Overview

This model uses sensor data to predict the failure or non-failure state of five components in a hydraulic system.

Input Features: The model is trained on data from the following sensors: CE, CP, SE, PS6, PS2, FS1, FS2, EPS1, TS4, TS2. Statistical features are calculated for each sensor, such as mean, standard deviation, maximum, minimum, range, and RMS values.

Output: The model predicts the states of five components: cooler, valve, pump, accumulator, and stable. The output is a classification indicating the operational state of each component.

Model Algorithm: The model uses a Random Forest Classifier wrapped in a MultiOutputClassifier to handle the multi-label classification task. This allows for simultaneous predictions of multiple components.

Feature Importance

The following features are the most important for predicting the state of each component:

  • Cooler: CP_max
  • Valve: PS2_std
  • Pump: FS1_rms
  • Accumulator: FS2_mean
  • Stable: FS1_mean

These features have been determined based on feature importance scores derived from the Random Forest model.
